2.	Публикация на портале отчетности

Макет дашборда «Статус исполнения плана эксплуатационной готовности»

Страница 1
 
1.	Панель заголовка, фильтров, сброса фильтров, даты отчета 

 
  
1.1.	 Состав фильтров:
•	Дата отчета – дата еженедельной выгрузки из графика 
•	Блок - переключает отображаемые данные на дашборде по соответствующему Блоку
•	Объект/СП - переключает отображаемые данные на дашборде по соответствующему Объекту/СП 
•	Сброс фильтров 

2.	Блок общих прогрессов 

 

Основой для измерения прогресса выполнения работ является график в MS Project. 
Значения в блоке остаются неизменными при применении фильтров и кросс-фильтрации на странице.
Для всех расчетов необходимо отфильтровать данные по критерию «Суммарная задача» = «нет»


2.1.	 Общие показатели
•	Общий прогресс план
•	Общий прогресс факт
•	Общее накопительное отклонение
•	Общий прогресс. План, % = Сумма запланированных мероприятий, которые должны быть завершены на дату отчета, шт./ Общее количество мероприятий, шт.* 100%
•	Общий прогресс. Факт, % = Сумма выполненных мероприятий на дату отчета, шт./ Общее количество мероприятий, шт.* 100%
•	Общее накопительное отклонение, % = Общий прогресс. Факт, % - Общий прогресс. План, %
2.2.	Общий статус выполнения мероприятий
•	Всего, шт.
•	План, шт.
•	Факт, шт.
•	Откл., шт.
•	Всего, шт. - Общее количество мероприятий, шт.
•	План, шт. - Сумма запланированных мероприятий, которые должны быть завершены на дату отчета, шт.
•	Факт, шт. - Сумма выполненных мероприятий на дату отчета, шт.

3.	Блок «Безопасность» 
 
Значения в блоке изменяются при применении фильтров из п.1, кросс-фильтрации в блоке  неактивна.
Для всех расчетов необходимо отфильтровать данные по критерию «Суммарная задача» = «нет», «Раздел» = «Безопасность»
Кнопка «Детализация» - по кнопке открывается лист «Безопасность – детализация»


3.1.	  Показатели прогресса по блоку «Безопасность»
 
•	Прогресс по разделу, план
•	Прогресс по разделу, факт
•	Накопительное отклонение по разделу
•	Прогресс по разделу. План, % = Сумма запланированных мероприятий, которые должны быть завершены на дату отчета в разрезе раздела, шт./ Общее количество мероприятий в разрезе раздела, шт.* 100%
•	Прогресс по разделу. Факт, % = Сумма выполненных мероприятий на дату отчета в разрезе раздела, шт./ Общее количество мероприятий в разрезе раздела, шт.* 100%
•	Накопительное отклонение по разделу, % = Прогресс по разделу. Факт, % - Прогресс по разделу. План, %
3.2.	Статус выполнения мероприятий

 

•	Всего мероприятий, шт - Общее количество мероприятий в разрезе раздела, шт.
•	Выполненные мероприятия, план, шт – плановое количество мероприятий, которые должны быть выполнены на дату отчета в разрезе раздела
•	Всего выполненных мероприятий, факт, шт – общее фактическое количество мероприятий, которые выполнены на дату отчета в разрезе раздела
•	в т.ч. кол-во выполненных мероприятий из плана, факт, шт – фактическое количество мероприятий из плана на отчетную дату, которые выполнены на дату отчета в разрезе раздела
•	в т.ч. кол-во выполненных мероприятий, факт, шт – фактическое количество мероприятий из графика, выполненных из будущих периодов в разрезе раздела
•	Откл., шт - разница Кол-во выполненных мероприятий из плана, факт, шт. – Кол-во выполненных мероприятий, план, шт.
•	Выполнение плана мероприятий, % - отношение Кол-во выполненных мероприятий из плана, факт, шт./ Кол-во выполненных мероприятий, план, шт.
•	Всего мероприятий в работе, шт - количество мероприятий со статусом «в работе»: в выгрузке из графика «Процент завершения» не равен 0% и не равен 100%, столбец ««Окончание» (дата окончания мероприятия) < Дата отчета
•	Мероприятия не начаты, шт - количество мероприятий со статусом «не начаты»: в выгрузке из графика «Процент завершения» равен 0%, столбец «Начало» (дата начала мероприятия) <= Дата отчета, столбец ««Окончание» (дата окончания мероприятия) > Дата отчета
•	Мероприятий будущих периодов, шт - количество мероприятий со статусом «не начаты»: в выгрузке из графика «Процент завершения» равен 0%, столбцы «Начало», «Окончание» (дата начала и окончания мероприятия) > Даты отчета

3.3.	 Кольцевой график «Статистика выполнения плана мероприятий»
 
100% для кольцевого графика - общее количество мероприятий по разделу
Общее количество мероприятий по графику состоит из: 
-	Выполненные мероприятия, шт./ % - общее количество выполненных мероприятий на дату отчета
-	Отставания, шт./ % - кол-во невыполненных мероприятий из плана на дату отчета
-	Мероприятия в работе, шт./ % - количество мероприятий со статусом «в работе»: в выгрузке из графика «Процент завершения» не равен 0% и не равен 100%, столбец «Окончание» (дата окончания мероприятия) > Дата отчета
-	Мероприятия не начаты, шт./ % - количество мероприятий со статусом «не начаты»: в выгрузке из графика «Процент завершения» равен 0%, столбец «Начало» (дата начала мероприятия) <= Дата отчета столбец «Окончание» (дата окончания мероприятия) > Дата отчета
-	Мероприятия будущих периодов, шт./ % – мероприятия, к которым не приступили (процент завершения = 0%), дата начала и окончания мероприятий > Даты отчета.

2.2.	 Анализ выполнения плана мероприятий по категориям
 

Всего, шт. - общее количество мероприятий по разделу в разрезе категорий
План, шт.  – количество мероприятий со сроком окончания <= Даты отчета
Факт от плана, шт.  – количество мероприятий со сроком окончания <= Даты отчета, «Процент завершения» равен 100%
Откл. от плана, шт. - Факт, шт. - План, шт.
Факт вне плана, шт.  - количество мероприятий со сроком окончания  >  Даты отчета, «Процент завершения» равен 100%
Факт всего, шт.  - Факт от плана + Факт вне плана

Сортировка задач в таблице по столбцу «Окончание»

условие: Откл. < 0
